# 個人記帳與消費分析系統

本專案是手機網頁版的個人記帳與消費分析系統，包含預算設定、可自訂分類預算百分比、消費紀錄、分類統計、圓餅圖、預算預警與 AI 智慧消費分析。

這個版本不使用 PHP，改用 Node.js 當後端代理 Gemini API。專案名稱已統一改為「個人記帳與消費分析系統」，不再使用「Gemini API Project」作為專案名稱。

## 為什麼需要 Node.js 後端？

不要把 Gemini API Key 直接寫在前端 `script.js`，因為使用者可以從瀏覽器開發者工具看到金鑰。這個版本的做法是：

```text
手機/瀏覽器網頁 → Node.js 後端 /api/analyze → Gemini API → 回傳 AI 分析結果
```

## 使用步驟

1. 安裝 Node.js 18 以上版本。
2. 把整個資料夾放到你想要的位置，例如桌面。
3. 複製 `.env.example`，改名成 `.env`。
4. 打開 `.env`，填入自己的 Gemini API Key：

```env
GEMINI_API_KEY=你的_Gemini_API_Key
GEMINI_MODEL=gemini-2.5-flash
```

5. 在資料夾內開啟終端機，執行：

```bash
npm start
```

或：

```bash
node server.js
```

6. 用瀏覽器打開：

```text
http://localhost:3000
```


## 新增功能：分類預算百分比

在「分類預算百分比設定」區塊中，可以自行設定餐飲、交通、娛樂、購物、學習、生活用品、醫療與其他的預算比例。系統會依照「可用消費預算」自動換算各分類可用金額，並在分析結果中顯示：

- 各分類設定比例
- 各分類預算金額
- 各分類已花金額
- 剩餘金額
- 是否超出分類預算

如果所有分類比例加總超過 100%，系統會顯示提醒，方便調整預算分配。

## 注意事項

- 不要直接雙擊 `index.html` 測試 AI 功能，因為這樣不會啟動 Node.js 後端。
- 如果 Gemini 回報模型不存在，可以把 `.env` 裡的 `GEMINI_MODEL` 改成你目前帳號可用的模型。
- 這個版本不需要 XAMPP，也不需要 PHP。

## AI 回覆長度

本版本已將 Gemini 回覆調整為簡短模式：

- AI 回覆限制為約 150 字以內
- `maxOutputTokens` 調整為 `400`
- Prompt 改為只回答「消費狀況、主要問題、改善建議」三點

如果想讓 AI 回答更完整，可以把 `server.js` 裡的 `maxOutputTokens` 改大，例如 `800` 或 `1200`。


## 本版新增功能

- 可在每個分類設定「預算比例 (%)」。
- 可在每個分類設定「警示門檻 (%)」，例如 80% 代表該分類花到預算的 80% 就會提醒。
- 新增消費後，如果該分類達到警示門檻或超出分類預算，系統會主動跳出提醒視窗。
- 消費分析結果會顯示各分類的預算、警示金額、已花金額、剩餘金額與狀態。
